# 🛡️ Windows 应急响应基本指南（蓝队实战版）

------

# 一、事件背景与目标

**典型场景：**

- 上级通报存在恶意通信（如银狐木马）
- 已知：存在外联行为
- 未知：感染主机 / IOC 类型（IP / 域名）

**核心目标：**

1. 定位感染主机
2. 确认持久化机制（服务 / 计划任务等）
3. 找到恶意程序
4. 阻断与清除

------

# 二、总体响应思路（四阶段模型）

### 1️⃣ 发现（Detection）

- 外部通报 / 流量异常 / 告警

### 2️⃣ 定位（Identification）

- 找到具体受感染主机

### 3️⃣ 分析（Analysis）

- 找持久化机制 + 恶意文件

### 4️⃣ 处置（Response）

- 清除 + 加固 + 溯源

------

# 三、第一阶段：网络侧定位（优先级最高）

------

## 3.1 防火墙日志分析

### 必备条件

- 会话日志（源IP / 目的IP / 时间）
- NAT 映射（内网IP ↔ 外网IP）

------

## 3.2 排查方法

### ✔ 已知恶意 IP

直接检索：

```text
目的IP = IOC_IP
```

------

### ✔ 已知恶意域名

检索：

- URL日志
- DNS日志

------

### ✔ IOC 不明确（最常见）

重点筛选：

#### 异常通信特征

- 周期性连接（心跳）
- 小流量高频通信
- 非常规端口（如 4444 / 8080 / 53 TCP）
- 境外 IP（VPS 段）

------

## 3.3 DNS 日志分析（关键补充）

重点关注：

- 新注册域名
- DGA（随机字符串域名）
- 查询失败率高
- TXT记录通信

------

## 3.4 结果

👉 输出：

- 可疑内网IP（感染主机候选）

------

# 四、第二阶段：主机侧快速排查

------

## 4.1 网络连接定位进程

```powershell
netstat -ano
```

👉 目标：

- 找外联IP对应的 PID

------

## 4.2 进程分析

```powershell
tasklist /svc
```

关注：

- 异常路径进程
- 非系统目录运行的程序

------

# 五、第三阶段：持久化机制排查（重点）

------

## 5.1 服务（Service）

### ✔ 查询创建记录（最关键）

```powershell
Get-WinEvent -FilterHashtable @{LogName='System'; ID=7045}
```

------

### ✔ 判断依据

#### 异常特征：

- 服务名伪装或随机
- 可执行路径异常：

```text
C:\Users\
C:\ProgramData\
C:\AppData\
```

------

## 5.2 计划任务（高频手段）

------

### ✔ 查看任务

```powershell
schtasks /query /fo LIST /v
```

------

### ✔ 查看日志（最可靠）

日志路径：

```text
Microsoft-Windows-TaskScheduler/Operational
```

关键事件：

| ID   | 含义 |
| ---- | ---- |
| 106  | 创建 |
| 140  | 修改 |

------



```POWERSHELL
Get-WinEvent -FilterHashtable @{LogName='System'; ID=140} | Format-List *
```



### ✔ XML 分析（辅助）

重点字段：

- `<Command>`
- `<Arguments>`

------

### ⚠️ 注意

- XML `<Date>` 可伪造
- 不作为唯一依据

------

# 六、第四阶段：文件级取证（MFT 核心）

------

## 6.1 使用 NTFS MFT 分析

工具：

- MFTECmd（推荐）

------

## 6.2 核心原理

MFT 包含两套时间：

| 属性                        | 说明       |
| --------------------------- | ---------- |
| SI（$STANDARD_INFORMATION） | 易被篡改   |
| FN（$FILE_NAME）            | 更接近真实 |

------

## 6.3 关键判断

👉 若：

- SI ≠ FN

👉 结论：

🔥 时间被篡改（强可疑）

------

## 6.4 重点路径

### ✔ 计划任务

```text
C:\Windows\System32\Tasks\
```

------

### ✔ 木马常见落地路径

```text
C:\Users\*\AppData\
C:\ProgramData\
C:\Windows\Temp\
```

------

## 6.5 排查方法

筛选：

- FN 创建时间 ≈ 异常时间
- 可执行文件（.exe / .dll）
- SI/FN 时间不一致

------

# 七、第五阶段：关联分析（核心能力）

------

## 🔥 建立攻击链

将以下信息关联：

1. 外联IP（网络侧）
2. 可疑进程（主机侧）
3. 持久化机制（服务 / 任务）
4. 恶意文件（MFT）

------

## ✔ 命中以下条件可基本确认

- 新建 EXE（异常路径）
- 新建计划任务 / 服务
- 存在外联行为

👉 = 恶意程序（高置信度）

------

# 八、第六阶段：处置与加固

------

## 8.1 立即处置

- 隔离主机（断网）
- 停止恶意进程
- 删除：
  - 恶意程序
  - 服务
  - 计划任务

------

## 8.2 网络侧阻断

- 封禁 IOC IP / 域名

------

## 8.3 全网排查

- 同 IOC 横向搜索
- 检查是否存在横向移动

------

## 8.4 日志与证据保全

- 导出：
  - 安全日志
  - MFT
  - 内存（如有条件）

------

# 九、关键经验总结（实战要点）
